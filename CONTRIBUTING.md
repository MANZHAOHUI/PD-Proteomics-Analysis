# Contributing to PD Proteomics Analysis

Thank you for your interest in contributing to this project!

## How to Contribute

### Reporting Issues
- Use the GitHub issue tracker to report bugs
- Include detailed steps to reproduce the issue
- Provide information about your computing environment
- Include relevant error messages and logs

### Suggesting Enhancements
- Use the issue tracker to suggest new features
- Clearly describe the enhancement and its benefits
- Provide examples of how the enhancement would be used

### Code Contributions

#### Setting Up Development Environment
1. Fork the repository
2. Clone your fork
3. Create a new branch
4. Install dependencies (see README.md)

#### Coding Standards
- **Python**: Follow PEP 8 style guide
- **R**: Follow tidyverse style guide
- Add docstrings/comments to all functions
- Include unit tests for new functionality
- Ensure code runs on HPC environments

#### Commit Messages
- Use clear, descriptive commit messages
- Start with a verb in present tense
- Reference issue numbers when applicable

#### Pull Request Process
1. Update documentation for any changed functionality
2. Test your code thoroughly
3. Update CHANGELOG.md with your changes
4. Submit pull request with clear description
5. Link to any relevant issues

## Questions?
Open an issue with the "question" label or contact project maintainers.

Thank you for contributing!
